Accessibilità

Linee guida per garantire che l'applicazione sia usabile da tutti, inclusi gli utenti con disabilità.

Checklist WCAG AA

  • Contrasto: Tutti i testi mantengono un rapporto di contrasto minimo di 4.5:1 (WCAG AA)
  • Focus Visible: Ogni elemento interattivo ha un outline di focus chiaro (2px blu)
  • Keyboard Navigation: Tutti i componenti sono navigabili da tastiera (Tab, Enter, Esc)
  • ARIA Labels: Form inputs e button hanno aria-label descrittivi
  • Semantic HTML: Uso di tag semantici (header, main, nav, section, button)
  • Color Independence: Le informazioni non dipendono solo dal colore (es. badges hanno testo + colore)
  • Skip Links: Link per saltare ai contenuti principali (opzionale)
  • Error Messages: Messaggi di errore chiari e visibili, collegati ai campi
  • Form Labels: Ogni input ha una label associata con for e id
  • Modal Focus: Il focus è intrappolato nel modal e restituito all'apertura

Guida Integrazione

Come integrare questo style guide nel tuo progetto.

1. CSS Variables

I token CSS sono disponibili come variabili CSS nel file. Usa direttamente nei tuoi CSS:

.my-component {
  background-color: var(--color-bg-secondary);
  color: var(--color-text-primary);
  padding: var(--spacing-lg);
  border-radius: var(--radius-md);
  transition: all var(--transition-base);
}

2. Design Tokens JSON

Scarica i token JSON dal blocco script nel file HTML e importali nel tuo progetto:

// In JavaScript/Node.js
const designTokens = require('./design-tokens.json');

const primaryColor = designTokens.colors.primary;
const spacing = designTokens.spacing.md;

3. Copiare Componenti

Ogni componente ha il markup HTML e CSS completo. Copia il codice dalla sezione componenti e adattalo al tuo progetto.

4. Mantenere la Consistenza

  • Usa sempre le variabili CSS per colori, spacing e font size
  • Segui le linee guida di naming per le classi CSS
  • Mantieni gli stessi valori di contrasto per i colori
  • Applica le stesse transizioni e hover states
  • Rispetta le linee guida di accessibilità WCAG AA

5. Aggiornamenti Futuri

Se il design system viene aggiornato:

  1. Aggiorna le variabili CSS nel `:root`
  2. Aggiorna i design tokens JSON
  3. Rigenera questo documento con le nuove sezioni
  4. Comunica i cambiamenti al team
